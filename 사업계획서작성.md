# [SKILL] 사업계획서 작성 워크플로우

본 스킬은 사용자로부터 핵심 정보를 순차적으로 수집한 후, 고득점_사업계획서_작성_스킬.md를 기반으로 정부지원사업용 사업계획서를 자동 생성합니다.

---

## Ⅰ. 필수 정보 수집 (Interactive Questions)

사업계획서 작성 전 아래 12개 항목을 **순서대로** 사용자에게 질문하여 정보를 수집합니다.

### 📋 질문 목록

```
1. 과제명은 무엇인가요?
   예: "AI 기반 스마트 농업 관리 시스템 개발"

2. 개발 분야는 무엇인가요?
   예: ICT/AI, 바이오, 제조, 서비스, 콘텐츠 등

3. 대상 시장은 어디인가요?
   예: 국내 스마트팜 시장, B2B SaaS 시장 등 (규모/성장률 포함 시 가산점)

4. 핵심 기술은 무엇인가요?
   예: 딥러닝 영상 분석, 블록체인 인증, IoT 센서 융합 등

5. 경쟁사 대비 차별점은 무엇인가요?
   예: 특허 기술, 독자 알고리즘, 원가 절감 등

6. 목표 고객은 누구인가요?
   예: 30~40대 1인 가구, 중소 제조업체 등 (규모 수치 포함 시 가산점)

7. 예산은 얼마인가요?
   예: 정부지원금 5억 원 + 자부담 1억 원

8. 개발 기간은 얼마인가요?
   예: 12개월, 24개월 등

9. 팀 구성은 어떻게 되나요?
   예: CEO(경력), CTO(경력), 개발자 3명, 마케터 1명 등

10. 기존 성과 또는 보유 기술은 무엇인가요?
    예: MVP 개발 완료, 특허 2건, MoU 5건, 매출 1억 원 등

11. 필요한 인증/규격이 있나요?
    예: ISO 27001, GMP, KC 인증, FDA 승인 등

12. 추가 요구사항이 있나요?
    예: ESG 강조, 해외 진출 계획, 특정 양식 준수 등
```

## Ⅱ. 사업계획서 작성 프로세스

### [Step 1] 정보 수집 완료 확인

모든 12개 항목이 수집되었는지 확인합니다. 누락된 항목이 있으면 추가 질문합니다.

### [Step 2] 참고 문서 로드

다음 문서들을 참조하여 작성합니다:
- [사업계획서작성가이드.md](사업계획서작성가이드.md)
- [심사위원-페르소나.md](심사위원-페르소나.md)
- [IR.md](IR.md)
- 분야별 예제 문서

### [Step 3] P-S-S-T 프레임워크 기반 초안 작성

| 섹션 | 페이지 | 핵심 질문 |
|------|--------|-----------|
| **0. 요약** | 4p | 전체 사업 개요 (아이템 6줄 상세 요약) |
| **Ⅰ. Problem (문제인식)** | 6p | 왜 지금 이 사업이 필요한가? |
| **Ⅱ. Solution (실현가능성)** | 8p | 어떻게 문제를 해결할 것인가? |
| **Ⅲ. Scale-up (성장전략)** | 8p | 어떻게 돈을 벌고 시장을 장악할 것인가? |
| **Ⅳ. Team (팀 구성)** | 2p | 왜 우리가 이 일을 가장 잘할 수 있는가? |

#### 작성 원칙
- 4줄 이상의 개조식으로 상세히 작성
- 3000자 이상 3개의 토픽으로 구성
- 통계 출처와 세부 출처 반드시 기재
- '많은 고객' → '서울 거주 2030 여성 50만 명' 등 구체적 수치 사용

### [Step 4] 5인 심사위원 페르소나 기반 자체 평가

| 심사위원 | 평가 항목 | 배점 |
|----------|-----------|------|
| 김재훈 (기술) | 기술적 타당성, 혁신성, 차별화 | 20점 |
| 박정숙 (투자) | BM 수익성, 시장 진입 전략, ROI | 20점 |
| 이은지 (마케팅) | 타겟 고객 정의, 트렌드 부합성 | 20점 |
| 장민우 (법률/규제) | 규제 대응, 지적재산권 보호 | 20점 |
| 윤소영 (ESG) | 사회적 가치, 환경 영향, 지속성 | 20점 |

> **합격 기준:** 80점 이상 → 최종 제출 가능 / 80점 미만 → 보완 필요

### [Step 5] 피드백 반영 및 보완

- 80점 미만 항목에 대해 구체적 보완 사항 제시
- 데이터 보강, 경쟁사 분석 강화, 차별성 재강조

### [Step 6] 나노 바나나2 프롬프트 생성

1. **제품 컨셉 이미지 프롬프트** - 전체 사업 내용을 시각화
2. **웹 대시보드 프롬프트** - 관리자 화면 디자인용


### [Step 7] 모두의창업용 사업계획서 요약

1. **나의 아이디어를 한 줄로 소개해주세요** - 전체 사업내용을 한 줄로 소개
2. **아이디어를 떠올린 배경 이야기를 들려주세요** - 아이디어의 배경 3가지 토픽 1000자
3. **아이디어는 누구의 어떤 문제를 해결해주나요** - 아이디어의 문제해결 3가지 토픽 1000자
4. **아이디어를 어떻게 실현하고 싶으신건가요** - 아이디어의 실현 3가지 토픽 1000자

### [Step 8] IR용 슬라이드 자료

1. 생성된 사업계획서 문서를 IR.md를 사용해서 어울리는 디자인템플릿을 추가하여 IR용 슬라이드 자료를 생성합니다 

### [Step 9] IR 슬라이드별 나레이션 작성

1. 생성된 IR 슬라이드의 각 페이지 메시지에 맞춰 슬라이드별 발표 나레이션 파일을 작성합니다
2. 슬라이드 10장에 대해서 총 5분 이내로 맞춰주세요

### [Step 10] 질문응답 정리

1. 사업계획서 내용을 바탕으로 질문응답을 정리합니다 

### [Step 11] Humanize KR - AI 문체 다듬기

1. [CLAUDE.md](CLAUDE.md)의 5단계 파이프라인(탐지 → 윤문 → 감사 → 리뷰 → 판정)을 실행합니다.
2. 사업계획서 본문, 모두의창업용요약, IR 나레이션 문서에서 AI 특유의 번역투와 기계적 표현을 제거합니다.
3. 각 문서의 의미를 보존하면서 사람이 쓴 것처럼 자연스러운 최종본을 각각 생성합니다.

## Ⅲ. 사용 방법

### 🚀 시작 명령어

```
/사업계획서작성
```

또는

```
사업계획서 작성해줘
```

### 📝 워크플로우 실행

1. AI가 12개 질문을 순서대로 제시합니다
2. 사용자가 각 질문에 답변합니다
3. 모든 정보 수집 후 사업계획서 초안을 생성합니다
4. 5인 심사위원 관점에서 자체 평가 점수를 제시합니다
5. 80점 미만 시 보완 사항을 안내합니다
6. 최종 완성본과 나노 바나나 프롬프트를 제공합니다
7. 모두의창업용 사업계획서를 생성합니다
8. IR용 사업계획서 요약을 생성합니다
9. 생성된 IR 슬라이드에 맞는 슬라이드별 나레이션 파일을 생성합니다
10. 질문응답을 생성합니다
11. Humanize KR 파이프라인을 통해 최종 문체를 자연스럽게 다듬습니다
